# Security Policy

## Reporting a Vulnerability

Please do not report security issues through public GitHub issues.

Use one of the following private channels instead:

- GitHub Private Vulnerability Reporting for this repository, if enabled
- a direct private channel to the maintainers

When you report an issue, please include:

- affected commit, branch, or release
- impacted platform or toolchain
- clear reproduction steps
- proof-of-concept or minimal testcase when possible
- impact assessment

We will review reports on a best-effort basis and respond as soon as practical.

## Scope

Security reports are most useful when they involve:

- memory safety
- undefined behavior with security impact
- trust boundary violations in public parsing or execution surfaces
- unsafe default behavior in public APIs

## Coordinated Disclosure

Please allow maintainers time to validate the issue and prepare a fix before
public disclosure.
